Debugger protocol.

Paulo Moura
version:
1.0

2014/2/7
compilation flags:

(no dependencies on other files)

Public interface

reset/0

Resets all debugging settings, including spy points and context spy points.

compilation flags:
static
mode – number of solutions:
reset – one

debug/0

Starts debugging for all defined spy points and context spy points.

compilation flags:
static
mode – number of solutions:
debug – one

nodebug/0

Stops debugging for all defined spy points and context spy points.

compilation flags:
static
mode – number of solutions:
nodebug – one

debugging/0

Reports current debugging settings, including spy points and context spy points.

compilation flags:
static
mode – number of solutions:
debugging – one

debugging/1

True if the entity is being debugged.

compilation flags:
static
template:
debugging(Entity)
mode – number of solutions:
debugging(+entity_identifier) – zero_or_one

trace/0

Starts tracing all calls compiled in debug mode.

compilation flags:
static
mode – number of solutions:
trace – one

notrace/0

Stops tracing of call compiled in debug mode.

compilation flags:
static
mode – number of solutions:
notrace – one

spy/1

Sets a spy point for a predicate or a list of predicates.

compilation flags:
static
template:
spy(Predicate)
mode – number of solutions:
spy(+predicate_indicator) – one
spy(+list(predicate_indicator)) – one

spy/4

Sets a context spy point.

compilation flags:
static
template:
spy(Sender,This,Self,Goal)
mode – number of solutions:
spy(@term,@term,@term,@term) – one

nospy/1

Removes all matching predicate spy points.

compilation flags:
static
template:
nospy(Predicate)
mode – number of solutions:
nospy(@variable) – one
nospy(+predicate_indicator) – one
nospy(+list(predicate_indicator)) – one

nospy/4

Removes all matching context spy points.

compilation flags:
static
template:
nospy(Sender,This,Self,Goal)
mode – number of solutions:
nospy(@term,@term,@term,@term) – one

nospyall/0

Removes all spy points.

compilation flags:
static
mode – number of solutions:
nospyall – one

leash/1

Sets the debugger leash ports using an abbreviation (none, loose, half, tight, or full) or a list of ports (valid ports are fact, rule, call, exit, redo, fail, and exception).

compilation flags:
static
template:
leash(Ports)
mode – number of solutions:
leash(+atom) – one
leash(+list(atom)) – one
